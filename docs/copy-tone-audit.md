## 一、宏观层面

### 1.1 整体调性评估

当前文案处于「传统软件公司模拟器」到「AI Agent 黑心公司」的过渡状态。约 **40%** 文案已经触及 AI/LLM 主题（模型幻觉、GPU 挖矿、Prompt 工程等），但仍有 **60%** 停留在普通科技公司办公室喜剧的范畴。

好消息是：底层游戏机制（模型员工、Token 成本、技能树）天生适合 AI 工地风格，框架是对的，缺的是文案的「最后一公里」改造。

### 1.3 全局核心问题（4 个）

**问题 1：人类办公室梗过多**
大量事件围绕「茶水间八卦」「团建聚餐」「偷偷摸鱼」「办公室政治」展开，这些是人类科技公司的日常，不是 AI Agent 团队的日常。AI Agent 不会喝 LaCroix、不会在公园喂鸭子、不会因为薪资倒挂而愤怒。

**问题 2：缺少 AI/LLM 特有的灾难场景**
Token 耗尽、上下文窗口溢出、模型偏航、Tool Call 风暴、API 限流、缓存未命中、模型退化——这些 AI 特有的崩溃方式几乎没有出现在事故模板中。现在的「事故」换成任何人类程序员都成立。

**问题 3：员工事件（teamEvents）仍是传统 HR 剧本**
招聘、培训、聚餐、薪资——这些事件完全可以用 AI 语言重写。比如「性能评估」可以写成「Token 使用量审计」，「团建」可以写成「多模型协同蒸馏」。

**问题 4：Sprint 反馈/Boss 评语的 AI 感不足**
老板的吐槽停留在「工程师摸鱼」「写 Bug 多」的传统笑话上，缺少 AI 特有的黑色幽默。比如「你花的推理费用够买一辆特斯拉了」「这个 Agent 的 context window 比我的耐心还小」。

### 1.4 优先级排序（改动收益从高到低）

1. **🔴 紧急：员工事件（teamEvents.ts）**—— 最偏离调性的模块，全是传统 HR 剧本
2. **🔴 紧急：Sprint 反馈/Boss 评语（ResultReport.tsx）**—— 每次 Sprint 都出现，曝光率最高
3. **🟡 重要：事故模板（incidentTemplates.ts）**—— 核心玩法内容，替换纯人类梗
4. **🟡 重要：Game Over 文案**—— 曝光率高，容易产生记忆点
5. **🟢 常规：技能树（skillTrees.ts）**—— 名称和描述过于通用
6. **🟢 常规：策略描述（strategies.ts）**—— 少量润色即可
7. **🔵 可选：成就文案（achievements.ts）**—— 整体尚可，锦上添花
8. **🔵 可选：项目类型（sampleProjects.ts）**—— 已经很好了，微调即可

### 核心原则

改写时牢记三条红线：
1. **AI Agent 不需要"团建聚餐"**——它们需要的是推理配额和上下文清理
2. **AI Agent 不会"摸鱼喂鸽子"**——它们会 idle、会 loop、会 context window 溢出
3. **AI Agent 不"辞职"**——它们会被 shutdown、被替换版本、被蒸馏
