## 技术债务清单

| 优先级 | 问题 | 影响 |
|--------|------|------|
| 高 | 无后端服务 | 无法跨设备、无法多人 |
| 高 | 无 CI/CD | 手动部署、容易出错 |
| 中 | i18n 不完整 | 部分中文硬编码 |
| 中 | 无 E2E 测试 | 只有单元测试 |
| 低 | 无性能监控 | 不知道真实用户体验 |
| 低 | 无 Dockerfile | 部署不便 |

## 设计决策记录

### 为什么纯前端？
- 初期快速验证游戏玩法
- localStorage 足够 MVP 阶段
- 降低运维成本

### 为什么用 Vitest 而不是 Jest？
- Vite 原生支持，配置更简单
- ESM 优先，与 Vite 一致
- 更快的 HMR 集成

### 为什么移动端用原生交互而不是响应式？
- 用户明确要求"移动端适配不是 CSS 缩放"
- 底部导航栏、全屏弹窗面板、一屏信息原则
- 详见 `components/Mobile*.tsx`
