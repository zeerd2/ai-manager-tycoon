## 游戏玩法

### 基本规则

你经营一家 AI 软件开发公司，手下有一群性格各异的 AI 工程师。每轮 Sprint 你需要：

1. **选择团队成员** - 点击选择参与本轮 Sprint 的 AI 工程师
2. **选择项目** - 选择一个要推进的项目
3. **选择策略** - 选择开发策略（如"快稳狠"、"精雕细琢"等）
4. **执行 Sprint** - 点击 "Run Sprint" 查看结果

### 核心系统

| 系统 | 说明 |
|------|------|
| **团队成员** | 多位 AI 工程师，每位有独特的技能、薪资、心情和怪癖 |
| **项目系统** | 5 个难度等级（实习/普通/挑战/地狱/传说），每个项目有不同的难度、风险和进度要求 |
| **策略系统** | 6 种策略，影响进度、Bug、技术债和士气 |
| **事件系统** | Sprint 中可能随机触发搞笑事故（bug、过度设计、幻觉等），30+ 事故模板 |
| **组合事件** | 特定事件（如 bug + 过度设计）组合会触发连锁反应 |
| **稀有事件** | 3% 概率在 Sprint 中触发的特殊事件 |
| **员工关系** | 团队成员之间的关系评分（-100 ~ +100），影响协作效率（±20%） |
| **关系网络** | 可视化展示团队成员之间的情感关系 |
| **团队事件** | Sprint 前 15% 概率触发的团队社交事件，多选项影响士气、资金和进度 |
| **技能树** | 每个角色有独立的技能树，花费资金解锁技能，提升工程师能力 |
| **成就系统** | 16 项成就，涵盖员工、项目、经济、事故四大类别，支持进度追踪 |
| **稀有度体系** | 成就分为 common / rare / epic / legendary 四个等级 |
| **评分系统** | 根据项目完成数、Bug、技术债等综合评分（S/A/B/C/D/F） |
| **存档系统** | 支持 3 个手动存档位 + 自动存档，旧版存档自动迁移 |
| **自动存档** | 可配置自动存档开关和间隔时间 |
| **解锁机制** | 部分高级工程师需要完成一定 Sprint 数后才解锁 |

### 策略一览

| 策略 | 效果 |
|------|------|
| Move Fast & Break Things | 进度 ×1.5，Bug ×1.8，技术债 ×1.6 |
| Careful & Methodical | 进度 ×0.7，Bug ×0.4，技术债 ×0.5 |
| Pair Programming | 进度 ×0.8，Bug ×0.5，技术债 ×0.7，士气 +5 |
| Crunch Mode | 进度 ×1.8，Bug ×1.4，技术债 ×2.0，士气 -8 |
| YOLO Deploy | 进度 ×2.0，Bug ×2.5，技术债 ×2.2 |
| Refactor Sprint | 进度 ×0.2，Bug ×0.3，技术债 -1.0 |

### 游戏结束条件

- 公司破产（资金 ≤ 0）
- 所有可用员工 morale 归零（集体罢工）

## 成就列表

### Employee（员工类）
| 成就 | 条件 | 稀有度 |
|------|------|--------|
| 🎯 初露锋芒 | 完成第一个项目 | common |
| 🔥 满级大佬 | 培养全技能 100 的员工 | legendary |
| 👥 人才济济 | 解锁 6 名以上员工 | epic |
| 🌟 伯乐 | 培养总技能 ≥ 450 的明星员工 | epic |

### Project（项目类）
| 成就 | 条件 | 稀有度 |
|------|------|--------|
| 🚀 连中三元 | 单局完成 3 个项目 | rare |
| ⚡ 神速 | 在 5 个 Sprint 内完成项目 | epic |
| 💀 地狱难度通关 | 完成传说级（难度 ≥ 80）项目 | legendary |

### Economic（经济类）
| 成就 | 条件 | 稀有度 |
|------|------|--------|
| 💰 财务自由 | 剩余资金 ≥ 8000 | rare |
| 💸 散财童子 | 累计花费 ≥ 10000 | epic |
| 🏦 精打细算 | 只用最便宜员工完成项目 | rare |
| 🤑 投资回报 | 完成项目时剩余资金 ≥ 总资金的 80% | common |

### Incident（事故类）
| 成就 | 条件 | 稀有度 |
|------|------|--------|
| 🐛 Bug 制造机 | 单轮 Sprint 产生 20+ bugs | common |
| 💀 大难不死 | 在产生 15+ bugs 后完成项目 | epic |
| 📈 Murphy 定律 | 单局产生 50+ bugs | rare |
| 😵 全员崩溃 | 所有员工 morale 归零 | common |
| 🔁 铁人 | 同一员工连续参与 6 轮 Sprint | rare |
| 🧑‍💻 一人团队 | 单员工完成项目 | rare |
